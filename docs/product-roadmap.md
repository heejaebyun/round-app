# Round Product Roadmap

## Product Definition
- Round는 `Choice DNA 앱`이 아니라 `현실에서 갈리는 질문을 피드로 소비하고 이유까지 읽게 만드는 논쟁 피드 앱`이다.
- 핵심 루프는 `질문 → 선택 → 결과 → 이유 소비 → 다음 질문`이다.
- DNA는 메인 약속이 아니라 나중에 쌓이는 보상 레이어다.

## Phase 1. 홈 질문 피드 전환
- 첫 화면에서 바로 질문 1개가 보이게 한다.
- 선택 전에는 결과를 숨긴다.
- 첫 질문 선택률을 가장 중요한 초기 KPI로 본다.

## Phase 2. 선택 후 카드 확장
- 같은 카드 안에서 결과 퍼센트와 양 진영 이유를 연다.
- 선택 직후 `0.3초` 기대감 구간을 살리고, `near-miss` 메시지를 넣는다.
- KPI:
  - 이유 보기 클릭률 `30%+`
  - 다음 질문 이동률 `40%+`

## Phase 3. 이유 카드 → 쓰레드
- 이유는 카드형으로 소비하고, 깊은 대화는 바텀시트 쓰레드에서 본다.
- 댓글과 답글은 전부 `selected_option_id`를 끌고 들어간다.
- KPI:
  - 쓰레드 진입률
  - 이유/답글 작성률

## Phase 4. 질문 운영 OS
- 질문은 직접 창작보다 `현실 tension 번역`으로 운영한다.
- 최소 메타:
  - `topic`
  - `tension`
  - `split_score`
- 질문 생존 판정은 아래를 함께 본다.
  - 선택 수
  - 이유 클릭률
  - 답글률
  - 다음 질문 이동률
  - split score

## Phase 5. 행동 프로필
- 데이터가 충분히 쌓인 뒤에만 연다.
- 프로필은 테스트 결과가 아니라 행동 기록의 압축본으로 보여준다.
- DNA는 이 단계에서 보상 레이어로 다시 정리한다.

## This Week
1. 홈을 질문 피드형으로 더 정리한다.
2. 선택 후 카드 확장 보상을 강화한다.
3. 이유 카드에서 쓰레드로 진입하게 만든다.
4. 댓글/답글에 `selected_option_id`를 저장한다.
5. 질문 메타 `topic`, `tension`, `split_score`를 붙인다.

## Initial KPIs
- 첫 질문 선택률: `70%+`
- 이유 보기 클릭률: `30%+`
- 다음 질문 이동률: `40%+`
- 3문항 도달률: 핵심 유지 지표
