# Round 수익화 설계 (사업자 승인 후 적용)

## 원칙
- 질문 선택 리듬을 절대 깨지 않는다
- 첫 10문항은 광고 없음
- 리워드형 우선, 배너 비추천

## 광고 위치 (우선순위)

### 1순위: DNA 심화 보기 (리워드형)
- **위치:** DNA 프로필에서 "내 선택 패턴 더 보기" 전
- **방식:** 광고 시청 → 축별 상세 분석 해금
- **이유:** 유저가 "더 알고 싶다"는 동기가 가장 강한 시점

### 2순위: 전체 의견 심화 탐색 (리워드형)
- **위치:** 전체 의견 보기에서 답글/스레드 깊게 보기 전
- **방식:** 광고 시청 → 답글 전체 해금
- **주의:** 너무 일찍 걸면 이탈 증가

### 3순위: 모든 질문 완료 → DNA 진입 (전면형)
- **위치:** "오늘의 선택 완료" → DNA 보기 직전
- **방식:** 전면 광고 1회
- **주의:** 세션당 최대 1회

## 비권장 위치
- ❌ 질문 카드 사이
- ❌ 결과 화면 직후
- ❌ 첫 3문항 이내
- ❌ 이유 입력 직후
- ❌ 배너 광고 (몰입 파괴)

## 토스 미니앱 제약
- 토스 내에서는 외부 광고 네트워크 사용 불가
- 토스 자체 광고 SDK 또는 브랜드 콜라보만 가능
- `isTossMiniApp()` 기반 분기 유지

## 구현 순서 (사업자 승인 후)
1. 리워드형: DNA 심화 보기
2. 전면형: 질문 완료 후 1회
3. 브랜드 VS 카드 (Phase 2)
