# Round — 토스 미니앱 출시 로드맵

## 전략 요약

웹 MVP와 토스 미니앱을 동시에 진행한다.
코드베이스는 하나이고, 배포 채널은 두 개다.

```text
[하나의 코드베이스]
     ├── Vercel 배포 → 독립 웹 PWA (SNS 바이럴 유입)
     └── 토스 웹뷰 → 토스 미니앱 (대규모 유저 풀 유입)
```

## 핵심 원칙

1. 코드베이스는 하나다. 토스용과 웹용을 따로 만들지 않는다.
2. 웹은 제품 검증 채널이다. 광고 없이 핵심 루프와 바이럴을 본다.
3. 토스는 유저 확보 채널이다. 필요 시 토스 기능과 토스 인앱 광고를 붙인다.
4. 빠르게 출시하고 빠르게 측정한다. 완성도보다 루프 검증이 우선이다.
5. 질문 품질이 성패를 결정한다. 기술보다 질문 30개의 퀄리티가 중요하다.

## 비용 정리

| 항목 | 비용 |
|------|------|
| Vercel | 무료 시작 가능 |
| Supabase | 무료 시작 가능 |
| 토스 미니앱 입점 | 무료 |
| 도메인 | 연 1~2만 원 |
| 합계 | 사실상 0원에 가깝게 시작 가능 |

## 채널 역할

| 채널 | 역할 | 장점 |
|------|------|------|
| 토스 미니앱 | 유입 채널 | 대규모 사용자 풀, 앱 내 발견 가능성 |
| 독립 웹 PWA | 검증 채널 | 링크 공유 자유도, 브랜드 자산 축적 |

둘을 동시에 운영하면서 어디서 PMF 신호가 더 강한지 관찰한다.

## 메모

- 독립 웹은 광고 없이 유지한다.
- 토스는 필요 시 토스 인앱 기능과 수익화를 후순위로 붙인다.
- 첫 출시 목표는 수익화가 아니라 핵심 루프 검증이다.
